BLOCO 1: HUB OMNICANAL (RADIAL)

DNA DA CAMPANHA (ESCORE + ESCOPO)

Centro = decisão mínima que ativa todo o resto. Nodes = pilares que a campanha obrigatoriamente define.

  • Direção: interna / externa / híbrida
  • Experiência: digital / presencial / híbrida
  • Públicos & tiers: associados / comunidade médica / sociedade / convidados / imprensa
  • Janela: início, fim, prazos de inscrição, marcos
  • Resultados finais: (ex) aprovações em teste pós-treinamento; inscrições por tier; adesões públicas

Regra: se for híbrida, a camada interna existe sempre (kit + alinhamento + risco).

TRILHAS PARALELAS (INTERNA x EXTERNA)

Divide o que o time precisa receber/treinar vs. o que o público recebe/experimenta.

  • Critério de pronto: tracking + destino + follow-up definido

BLOCO 5: FLUXOGRAMA DE DECISÃO (DECISION TREE)

ÁRVORE-MÃE (SE → ENTÃO) — V2 EXTREMA

Decisão prática → abre módulos físicos/digitais/evento + define resultados verificáveis.

ÁRVORE-MÃE COMPLETA (SE → ENTÃO) · ativação automática de módulos
A) DIREÇÃO 
SE Direção = INTERNA:
  → definir: objetivo interno (treinamento / confraternização / alinhamento / operação)
  → definir: haverá físico? (impressos / uniformes / brindes / coffee / locação)
  → definir: validação (teste pós-treinamento? presença? checklist?)
SE Direção = EXTERNA:
  → definir: público(s) e segmentação (sociedade / associados / médicos / etc.)
  → definir: experiência (digital / presencial / híbrida)
SE Direção = HÍBRIDA:
  → SEMPRE criar CAMADA INTERNA:
      - kit interno (1-pager + FAQ + scripts)
      - orientação “o que falar / o que não falar”
      - escalonamento (quando acionar diretoria/jurídico)
B) EXPERIÊNCIA 
SE Experiência = PRESENCIAL ou HÍBRIDA:
  1) Acesso:
     - aberto / fechado / segmentado (tiers)
     - se segmentado → definir TIERS (associados, médicos, convidados, público, imprensa, parceiros)
       para cada tier: elegibilidade + validação + cota + janela + benefício
  2) Inscrição:
     - necessária? (sim/não)
     - plataforma (form/sympla/lista)
     - limite de vagas + lista de espera
     - prazo de inscrição (data/hora) + confirmação
     - check-in (sim/não) + credenciais (crachá/pulseira) + validação (lista/CRM/doc)
  3) Programação:
     - existe agenda? (sim/não)
     - blocos (n) + duração + Q&A
     - convidados? (sim/não) → papel + logística + termo de imagem
  4) Produção física (tudo que estiver ON vira tarefa + orçamento):
     - impressos (tipos + tiragem por tier + entrega)
     - brindes/kits (composição + qty por tier + montagem)
     - uniforme/identificação (qty/tamanhos)
     - coffee/alimentação (formato + qty + restrições)
     - locação/infra/A-V (capacidade + contrato + segurança + acessibilidade)
     - captação (foto/vídeo + entregáveis)
  5) Conversão/Coleta:
     - haverá cadastro/adesão no local? (sim/não)
     - QR/form/tablet + campos + LGPD
     - follow-up pós-evento (sim/não) + SLA

SE Experiência = DIGITAL ou HÍBRIDA:
  1) Frentes ativas (sim/não):
     - orgânico / ads / landing+form / WhatsApp / e-mail / PR / comunidades
  2) Estrutura:
     - peça única / série (n) / jornada (ordem) / always-on
  3) Conteúdo:
     - natureza: técnico / institucional / educativo / cultural / acadêmico / político-social
     - tom: institucional/didático/técnico/mobilizador + lista do que NÃO pode soar
  4) CTA & Destino:
     - ação: cadastro / associação / inscrição / contato WhatsApp / download
     - destino: landing / WhatsApp / form / evento
  5) Tracking:
     - UTM + eventos (view/submit/whats click)
     - coleta de dados (sim/não) → LGPD
  6) Follow-up:
     - quem responde? (nome/equipe)
     - SLA (ex: resposta em 4h/24h)
     - scripts + respostas rápidas
C) ORÇAMENTO (R$) — sem “baixo/médio/alto” 
Definir faixa total disponível (R$): 0–1k | 1–5k | 5–20k | 20–100k | 100k+
Definir tetos por linha (R$):
  - criação/produção | mídia | impressos | kits/brindes | uniforme/ID | coffee | locação/infra | A/V | captação | plataforma inscrição | logística | contingência
Regra: módulo ON sem linha de custo (R$) = campanha NÃO pronta.
D) “FINALMENTES” (RESULTADOS FINAIS) — exemplos específicos 
Escolher 1–3 resultados finais (com meta numérica + janela + custo por resultado):

INTERNO:
  - % do time com treinamento concluído até data X
  - % aprovado em teste pós-treinamento (nota mínima Y)
  - redução de erros operacionais (baseline vs pós)

PRESENCIAL:
  - inscrições por tier (n) + taxa de comparecimento (%)
  - nº de adesões públicas no local (n)
  - nº de prescritores cadastrados com CRM validado (n)

DIGITAL:
  - nº de cadastros completos (n)
  - nº de conversas qualificadas no WhatsApp (n) + SLA cumprido (%)
  - taxa landing→ação (%) + custo por ação (R$)

Regra: “resultado final” precisa ser verificável (contável/auditável).
